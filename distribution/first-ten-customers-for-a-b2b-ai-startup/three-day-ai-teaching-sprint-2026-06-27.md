# Three-day AI teaching sprint - 2026-06-27

Prepared: 2026-06-27.

Purpose: recover pace by focusing the campaign on practical AI teaching across X and LinkedIn for the next three days, while avoiding open-ended browser and token burn.

## Strategy

The main audience is people trying to become useful with AI at work.

Core topics:

- agents and workflows
- Claude Code / Cursor / AI coding systems
- prompt-to-system thinking
- model and tool choice
- evals, harnesses, acceptance tests, and review loops
- new AI tools and updates translated into plain English
- occasional AI jokes or anti-slop takes when they teach a real distinction

Product management, startups, and GTM are bridge examples only.

## Daily cadence

X:

- 2 native posts per day.
- 2 high-engagement reply batches per day.
- Each reply batch is 6-7 useful no-link replies.
- Every X reply must be drafted at 240 characters or fewer before posting. Do not rely on last-second trimming in the composer.

LinkedIn:

- 1 native post per day.
- 2 high-engagement comment batches per day.
- Each comment batch is 6-7 useful no-link comments.
- If LinkedIn native posting fails because the composer is flaky, do not burn the block. Log the failed attempt, then use that time for one extra high-quality LinkedIn comment.

4-hour signal checks:

- Check X and LinkedIn notifications/comments first.
- Reply to substantive inbound comments before posting anything new.
- Refresh only posts touched in the last 48 hours or posts that likely have new replies.
- Log replies, profile movement, and visible counters when evidence is available.

## Token and time budget

Each 4-hour heartbeat should stop after one of these is true:

- one verified public action is posted and logged,
- all substantive inbound comments are answered and logged,
- 30 minutes of browser work is reached,
- two searches per platform have failed to surface a strong target.

Do not repeatedly refresh metrics just to watch numbers move. Metrics are useful after a new public action, a likely reply, or the daily review block.

Maintenance checks can stop after one verified public action. Active sprint blocks cannot. In an active sprint block, the floor is:

- 6-7 X replies if running an X reply batch,
- 6-7 LinkedIn comments if running a LinkedIn comment batch,
- or both if the block is explicitly a full distribution block.

Stop early only when the browser path fails, the platform blocks input, or two searches per platform fail to surface rooms that pass the scorecard.

## Reach math

100,000 monthly reach needs about 3,333 reached impressions per day.

One comment is not enough. Even 10 comments are not enough if the rooms are weak.

Daily target during this sprint:

- 12-14 X replies,
- 12-14 LinkedIn comments,
- 2 X native posts,
- 1 LinkedIn native post or a verified LinkedIn comment fallback.

If the average comment/reply gets 100 views, 24-28 comments/replies create 2,400-2,800 reached impressions/day before owned posts. That is still below the 100k path, so every batch needs at least 2-3 strong rooms with real distribution: high-view X posts, high-reaction LinkedIn posts, high-comment threads, or posts where known AI builders/operators are already talking.

## Review after day 3

On 2026-06-30, review:

- which channel generated replies, not only views,
- whether X native posts or X replies produced more profile movement,
- whether LinkedIn native posts worked reliably or comments are still the better path,
- which topic produced the best replies: agents, AI coding, prompts-to-systems, evals, tools/updates, or humor,
- whether to increase volume, change hooks, or narrow to the strongest lane.

The default adjustment rule:

- If comments/replies create replies, increase room entry.
- If native posts create profile movement, increase native posting.
- If neither creates replies, sharpen the topic around a single painful confusion instead of broad AI teaching.
