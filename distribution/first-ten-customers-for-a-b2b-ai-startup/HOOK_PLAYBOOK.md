# Hook Playbook — first lines that make the TG stop, read, and stay

The hook is the first line: it shows above the fold, it IS the post on the timeline. Its job is self-recognition: the reader sees their own Tuesday in it. Curiosity without recognition is clickbait; recognition without payoff is complaining. We need both.

## The 8 hook shapes (rotate; never two consecutive posts with the same shape)

| Shape | Formula | Example |
|---|---|---|
| **The mirror** | Describe their exact moment of pain, second person, present tense | "You rewrote the prompt eleven times this month. Quality moved five percent." |
| **The receipt** | Lead with a specific result + the unlikely move that caused it | "Error rate fell from 9% to 2% the day we deleted the planner." |
| **The confession** | First-person mistake with a cost attached | "We spent nine days debugging an agent. The bug was a pagination default." |
| **The tiny test** | A check they can run in minutes, promised up front | "Pull your agent's last 50 runs. This 5-minute check tells you if you even need it." |
| **The myth flip** | Name the popular belief, show the receipt against it | "Everyone says better prompts. The biggest jump we ever saw came from one attached example." |
| **The costly default** | A setting/habit that silently burns money or trust | "One retry setting was 30% of our token bill. Nobody had ever looked at it." |
| **The scene** | Drop them into a concrete moment, mid-story | "Week three. The PM asks how we'll know if the model got worse. Silence." |
| **The dry observation** | Comic register: the absurdity they live daily | "The agent added a feature nobody asked for, wrote tests for it, and called the task complete." |

## Hook rules

1. First line ≤ 100 characters. It must survive alone on a timeline.
2. Name the pain or the payoff in the first line. Both by line two.
3. Concrete beats clever: a number, a named artifact (prompt, repo, dashboard), or a timeframe in every hook.
4. Never promise what the post doesn't deliver. Curiosity gap yes, bait-and-switch never.
5. Second person ("you") or first person ("we/I"). Never "people/teams/everyone" in the hook: that's a lecture, not a mirror.
6. No questions as hooks more than once per week. Statements carry authority.

## Comprehension rules (easy to understand, skill actually transfers)

1. One idea per post. If the post teaches two things, it's two posts.
2. Any term of art gets a plain-words gloss in ≤ 6 words, inline: "drift (the output changing quietly)".
3. Steps are numbered and start with verbs. A busy founder should be able to DO the thing today.
4. Reading level: explain like the reader is smart but distracted. Short sentences. No nested clauses in key lines.
5. Every post ends with the skill increment stated plainly: the test to run, the rule to adopt, the artifact to create.
6. The save test: would a founder screenshot this to use next sprint? If not, sharpen the payoff.

## Wiring

- Each bank day maps to AUDIENCE_MAP.md curriculum items; the hook shape rotates per the table above.
- Hook shape used per day is tracked in the bank README index to enforce rotation.
