# X Premium operating plan

Created: 2026-06-30T13:20:00+05:30.

Context: X Premium is now active / in verification review for the account. Use Premium to increase useful X surface area, not to create longer slop.

Source checked: X Help Center, "About X Premium" (`https://help.x.com/en/using-x/x-premium`), reviewed 2026-06-30.

## Useful Premium levers

X documents these Premium features as relevant to this campaign:

- reply prioritization, with larger prioritization on higher tiers,
- blue checkmark after eligibility review for Premium/Premium+,
- longer posts and replies up to 25,000 characters,
- 1-hour edit window for original posts and quote posts,
- bookmark folders,
- reduced ads on Premium, no ads / Radar Search / Articles on Premium+,
- verified-only reply access after verification,
- Media Studio and Articles for Premium/Premium+.

## Campaign changes

### 1. X becomes the faster testing channel

Daily X target increases from "keep up with LinkedIn" to "use Premium surface area aggressively":

- 2-3 native X posts/day.
- 3 X reply batches/day remains the floor.
- Each X reply batch remains 5-6 verified replies.
- When browser stability allows, add one extra opportunistic X mini-batch of 3 verified replies in a Premium-relevant room.

LinkedIn remains first-class, but X is now the faster idea lab.

### 2. Use longer posts selectively

Do not turn every X post into a long post.

Use longer X posts only for:

- a 600-1,200 character mini-essay,
- a practical checklist,
- a "workflow you can copy today",
- a strong reply that needs more nuance than 240 characters,
- a quote post with a concrete framework.

Default X native post length should still be 120-280 characters.

New daily content mix:

- one short sharp X post,
- one practical X mini-framework,
- optional third X post only if it is based on live signal from replies/comments.

### 3. Keep reply quality high

Even with longer replies available, most replies should stay short.

Rules:

- normal X replies: 180-240 characters,
- high-context replies: 240-600 characters only when the target is strong and the point needs nuance,
- no generic agreement,
- no links unless directly asked,
- every reply must add a frame, caveat, workflow step, or test.

### 4. Exploit reply prioritization

Premium reply priority should be used where ranking matters:

- verified-heavy posts,
- medium-hot posts with 30-500 likes and under 80 replies,
- posts from AI/tool/product accounts where comments are still readable,
- verified-only posts once the account is fully verified,
- new tool/update posts where early comments shape the conversation.

Avoid wasting priority on:

- mega-viral posts with thousands of replies,
- meme replies with no audience fit,
- political/news hijacks,
- low-quality "AI prompts" engagement bait.

### 5. Use bookmarks as an operating system

Create bookmark folders manually in X if available:

- `AI workflow targets`
- `Agent/eval targets`
- `Cursor/Claude targets`
- `Post ideas`
- `Respond later`

During scouting, bookmark strong targets instead of losing them. During scheduled blocks, work from saved targets before running new searches.

### 6. Use edit window carefully

Premium allows editing original posts and quote posts for a limited window. Use it for:

- typos,
- unclear phrasing,
- broken formatting,
- adding one missing line to sharpen the point.

Do not use edits to materially change a post after people engage with it.

Replies are still treated as final unless X clearly exposes edit support for that specific reply.

### 7. Do not edit profile during checkmark review

While the blue checkmark is under review or newly applied, do not change:

- profile photo,
- display name,
- handle.

X warns that these changes can affect checkmark validation. Profile edits remain blocked unless the user explicitly approves that exact action and accepts the checkmark-review risk.

## Premium search lanes

Use the 30-lane X Premium search bank in `search-queues.md` first during X batches. The old behavior of repeating 3-5 generic searches is too narrow and leaves reach on the table.

Batch search routine:

1. Start with the lane family that best matches the current topic or live page.
2. Sample at least 3 different search lanes before concluding that no strong room is available.
3. In each full X reply batch, include targets from at least 2 lane families when possible.
4. If a lane produces spam or low-quality tool launches twice, mark it cold for the day and switch.
5. Save good targets into bookmarks/target notes before replying if the editor is unstable.
6. Track which lane family produced likes/replies/profile movement within 24 hours.

Default lane families:

- operator AI at work,
- beginners and learning,
- agents/evals/harnesses,
- AI coding workflows,
- models/tools/updates,
- PMs/founders/operators/memes.

Legacy quick searches, still useful when the search bank is unavailable:

- `("Claude Code" OR Cursor OR "AI workflow" OR "AI agents") min_faves:30 since:YYYY-MM-DD`
- `("context engineering" OR "prompt engineering" OR "AI agents") min_faves:30 since:YYYY-MM-DD`
- `("AI at work" OR "AI productivity" OR "AI automation") min_faves:30 since:YYYY-MM-DD`
- `("model choice" OR "evals" OR "coding agents") min_faves:30 since:YYYY-MM-DD`
- `from:cursor_ai OR from:AnthropicAI OR from:OpenAI OR from:github "agent" OR "workflow"`

When Premium+ Radar is available, use Radar for:

- trend detection around Cursor, Claude Code, agents, evals, and AI productivity,
- finding fast-rising posts before they cross 500 likes,
- identifying authors whose posts create replies rather than only impressions.

## Premium KPI changes

Track X separately from LinkedIn:

- verified X replies posted,
- replies that receive likes/replies within 24 hours,
- profile followers and 7-day impressions,
- native X post views at 1h, 4h, 24h,
- whether longer posts outperform short posts,
- which target room type produces replies.

Do not treat Premium as a guarantee. It is only leverage if the comments are earlier, sharper, and placed in better rooms.
