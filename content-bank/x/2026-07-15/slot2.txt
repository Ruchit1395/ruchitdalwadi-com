A 20-minute tool audit for any agent that's misbehaving:

1. Pull the logs for the last 20 runs. Count which tools were actually called. There's usually a long tail of never-used tools — remove them. They're not free: every tool description eats context and adds a wrong option at each decision.

2. For each surviving tool, check that the description answers one question: when should the model pick THIS over its neighbors? "Searches the web" is a label. "Use for facts newer than 2025 or anything with a price" is a decision rule.

3. Find the overlap pairs — two tools that could both handle a task. Merge them, or give one an explicit boundary. Overlap is where agents flip-flop.

Agents rarely need more capability. They need fewer, sharper choices.
