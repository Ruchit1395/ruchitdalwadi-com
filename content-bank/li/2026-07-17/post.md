Two teams adopt the same AI coding tool. Six weeks later, one ships noticeably faster and the other is drowning in unreviewed diffs, quietly concluding the tool "isn't ready."

I keep seeing this split, and it's never about the tool. It's the loop around it.

The fast team writes a two-minute brief before every session: the goal, the test that proves it, which files are in bounds. They run the test before reading the diff — failing code doesn't deserve review time. And when the tool repeats a mistake, they turn it into a project rule so next month starts smarter.

The slow team types a wish into a chat box and reviews whatever comes back.

The uncomfortable part: this is a management skill, not a technical one. Scoping, trust calibration, feedback loops — the same things that make delegation to people work.

Where does your loop leak — the brief, the review, or the follow-through?
